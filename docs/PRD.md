# Product Requirements Document (PRD) - Video Upload MVP

## 1. Introduction
This document outlines the requirements for a Minimum Viable Product (MVP) video application. The goal is to demonstrate a simple solution for video uploads and storage, targeting a presentation to the Board of Directors.

## 2. Objectives
-   **Primary Goal:** Create a simple web application to upload and list videos.
-   **Key Metrics:** Successful demonstration of the flow (Upload -> List).
-   **Deadline:** End of Week (EOW).

## 3. Functional Requirements

### 3.1. Video Upload
-   **FR-1:** The application must allow users to upload a video file.
-   **FR-2:** Uploaded videos must be stored in **Google Cloud Storage (GCS)**.

### 3.2. User Interface
-   **FR-3:** **Video List:** A webpage must list all available videos stored in the system.

## 4. Technical Constraints & Stack
-   **Cloud Provider:** Google Cloud Platform (GCP).
-   **Storage:** Google Cloud Storage (GCS).

## 5. Assumptions
-   The user has the necessary GCP permissions and credentials.
